# 1. Product Overview

DreamLoop is a **shared AI-generated adventure experience for two people**.

Two users connect and participate in an **infinite AI-generated story world**.

Each story event presents choices to both users. Their decisions shape the narrative.

The app focuses on:
- long-distance couples
- long-distance friends
- shared storytelling
- asynchronous gameplay

The product must feel:

- cute
- cozy
- adventurous
- occasionally mysterious or spooky

The experience should feel like a **shared evolving dream world**.

---

# 2. Core Product Goals

The app must:

1. Create shared experiences for remote users.
2. Encourage daily engagement.
3. Deliver AI-generated interactive stories.
4. Support asynchronous participation.
5. Be simple enough for rapid iteration.

The initial release should prioritize **speed of development and simplicity**.

# 5. AI System

AI will generate story events.

AI provider:

OpenAI API (or compatible LLM)

AI will generate:

- story event descriptions
- decision options
- narrative progression

AI should output structured responses.

Example format:

{
  event: "A glowing cave appears near your village.",
  options: [
    "Enter the cave",
    "Camp outside",
    "Send a drone inside"
  ]
}

AI prompts must be optimized for:

- short responses
- creativity
- narrative consistency

---

# 6. Core Gameplay Loop

The game loop operates as follows:

1. Two users create or join a session.
2. A story event is generated by AI.
3. Both users receive the event.
4. Each user chooses an option.
5. Once both choices are submitted:
   - next event is generated
6. Story continues indefinitely.

This loop is the **core mechanic of the product**.

# 7. User Journey

## Step 1 — Install App

User installs DreamLoop.

---

## Step 2 — Authentication

User signs in using:

- Apple
- Google

---

## Step 3 — Invite Partner

User invites a partner using:

- shareable invite link
- invite code

The partner joins the same session.

---

## Step 4 — Session Creation

A shared session is created containing:

- two users
- session state
- story history

---

## Step 5 — First Story Event

The AI generates the first event.

Example:

"A glowing cave appears in the forest."

Options:

- Enter the cave
- Camp outside
- Send a drone inside

---

## Step 6 — Decision Phase

Each player submits a choice.

Players do not need to respond simultaneously.

The system waits until both choices are submitted.

---

## Step 7 — Story Resolution

When both choices are submitted:

- the story progresses
- the next event is generated

# 10. Session Logic

A session represents a shared story between two users.

Session state includes:

- active story event
- choices made
- previous events

The session continues until users leave or reset.

---

# 11. Story History

Each session should store previous events.

This allows:

- story continuity
- potential AI memory usage later

For MVP the history can be simple.
