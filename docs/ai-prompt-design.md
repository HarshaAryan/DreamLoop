# DreamLoop AI Prompt Design

This document describes how AI-generated stories should behave.

---

# Story Structure

Each story event contains:

- situation
- choices
- scene outcome

Stories should remain short and readable.

**Crucial Note:** There is NO prefixed story or predetermined ending. The story must remain highly flexible. It can have completely different endings based entirely on the choices made. A single story arc may span 3-4 days of real-world interaction before resolving. Every single action must be stored in memory to build the future plot.

---

# Event Format

Every event should include:

Title  
Description  
Choices  

Example:

Title:
The Hidden Cave

Description:
You notice a glowing cave behind a waterfall.

Choices:
Enter the cave
Camp outside
Send a drone inside

---

# Tone Variation

Stories should rotate between tones.

Cute
Adventurous
Romantic
Mysterious
Scary (rare)

---

# Tone Balance

Suggested distribution:

Cute: 40%  
Adventure: 30%  
Romantic: 15%  
Mysterious: 10%  
Scary: 5%

Rare scary events increase excitement.

---

# Story Continuity

Stories should reference past events.

Example:

"If the players saved a dragon earlier, the dragon may return later."

This builds a living narrative.

---

# Event Length

Events should be brief.

Users read them in notifications.

Ideal length:

1–3 sentences.

---

# Choice Design

Choices should feel meaningful. They should present "hard choices" that reveal the true nature and real-world personality of the human players.

Avoid obvious correct answers.

Choices should range across:
- bold adventures
- psychological or horror-based decisions
- emotional impacts and sacrifices

Choices should feel different.

Example:

Fight the dragon (Risk/Adventure)
Talk to the dragon (Empathy/Bonding)
Hide from the dragon (Caution/Horror)

---

# Surprise Factor

Occasionally include unexpected twists.

Examples:

- portal appears
- character gains magical ability
- creature becomes a companion

These moments create delight.

---

# Emotional Events

Not all events should be dramatic.

Quiet moments are important.

Examples:

- sitting by a fire discussing a difficult memory
- comforting each other after a scary event
- watching shooting stars
- resting by fire
- listening to rain

These moments strengthen emotional bonds and act as deep "bonding sessions" within the story.

---

# Episode System

Stories can occasionally form mini arcs.

Example:

Episode 1 — The signal  
Episode 2 — The cave  
Episode 3 — The dragon temple

Episodes make the world feel larger.
